# ربات ناشناس تلگرام + پنل مدیریت

یک ربات تلگرام شبیه NGL: هر کاربر با زدن `/start` یه لینک شخصی می‌گیره، هر کی اون لینک رو باز کنه
می‌تونه براش پیام ناشناس بفرسته. هویت فرستنده هیچوقت به گیرنده نشون داده نمیشه.
یک پنل مدیریت (وب) هم داره که ادمین می‌تونه همه‌ی پیام‌های رد و بدل شده رو ببینه، جستجو کنه و فیلتر کنه.

## تکنولوژی‌ها
- **Bot:** Node.js + [Telegraf](https://telegraf.js.org/)
- **Server:** Express
- **DB:** PostgreSQL (پلاگین Railway)
- **Session:** ذخیره‌شده در PostgreSQL (پایدار، بین ری‌استارت‌ها پاک نمیشه)

## قابلیت‌های جدید
- **آیدی اختصاصی کاربر:** هر کاربر با دکمه‌ی «✏️ تغییر آیدی» یا دستور `/setid` می‌تونه به‌جای عدد، یه آیدی دلخواه (حروف/عدد/آندرلاین) برای لینکش بسازه و هروقت خواست عوضش کنه.
- **منوی دکمه‌ای پایین چت:** «🔗 لینک من»، «✏️ تغییر آیدی»، «📖 راهنما» — همیشه در دسترسن، نیازی به تایپ دستور نیست.
- **مشخصات کامل فرستنده برای ادمین:** روی هر پیام توی پنل، دکمه‌ی «👤 مشخصات فرستنده» هست که آیدی عددی تلگرام (قابل کپی برای گزارش به پشتیبانی تلگرام)، یوزرنیم، نام کامل، زبان اکانت و وضعیت پرمیوم رو نشون میده. این برای رسیدگی به شکایت مزاحمته.

> **محدودیت مهم:** تلگرام شماره تلفن کاربر رو در اختیار بات نمی‌ذاره مگر خود کاربر با یک دکمه‌ی رسمی «اشتراک تماس» صریحا قبول کنه. این پروژه شماره تلفن جمع‌آوری نمی‌کنه چون هم خلاف قوانین تلگرامه هم اعتماد کاربرها رو می‌شکنه. آیدی عددی تلگرام (که همیشه در دسترسه) برای شناسایی حساب نزد پشتیبانی تلگرام کاملا کافیه.

## مرحله ۱: ساخت بات در تلگرام
۱. توی تلگرام برو سراغ [@BotFather](https://t.me/BotFather)
۲. بزن `/newbot` و اسم و یوزرنیم بات رو بده
۳. توکنی که میده رو نگه دار (بعدا لازمش داری)

## مرحله ۲: آپلود کد روی گیت‌هاب
```bash
cd anonymous-bot
git init
git add .
git commit -m "اولین نسخه ربات ناشناس"
git branch -M main
git remote add origin https://github.com/<username>/<repo-name>.git
git push -u origin main
```

## مرحله ۳: دیپلوی روی Railway
۱. برو [railway.app](https://railway.app) و با گیت‌هاب لاگین کن
۲. **New Project → Deploy from GitHub repo** و ریپازیتوری‌ای که ساختی رو انتخاب کن
۳. توی همون پروژه، **New → Database → Add PostgreSQL** بزن (این خودش `DATABASE_URL` رو می‌سازه و به سرویس بات وصل می‌کنه — لازم نیست دستی چیزی بنویسی)
۴. برو توی تنظیمات سرویس بات (نه دیتابیس) → تب **Variables** و این‌ها رو اضافه کن:

| متغیر | مقدار |
|---|---|
| `BOT_TOKEN` | توکنی که از BotFather گرفتی |
| `ADMIN_USERNAME` | یوزرنیم دلخواه برای ورود به پنل |
| `ADMIN_PASSWORD` | رمز عبور قوی و دلخواه |
| `SESSION_SECRET` | یک رشته تصادفی طولانی (مثلا از [این سایت](https://generate-secret.vercel.app/32)) |

> `DATABASE_URL` رو دست نزن — Railway خودش وصلش می‌کنه چون Postgres رو به همین پروژه اضافه کردی.
> `WEBHOOK_URL` رو هم لازم نیست بدی، کد خودش از `RAILWAY_PUBLIC_DOMAIN` استفاده می‌کنه.

۵. توی تب **Settings** سرویس، مطمئن شو **Public Networking** فعاله (یه دامنه‌ی `xxx.up.railway.app` بهت میده)
۶. صبر کن Deploy تموم بشه — توی لاگ‌ها باید ببینی: `🤖 Webhook ست شد` و `🚀 سرور روی پورت ... بالا اومد`

## مرحله ۴: تست
- توی تلگرام برو سراغ باتت و بزن `/start` — باید لینک شخصیت رو بگیری
- اون لینک رو با گوشی دیگه یا اکانت دیگه باز کن و یه پیام بفرست
- باید توی چت اول به صورت ناشناس دریافتش کنی

## مرحله ۵: ورود به پنل مدیریت
برو به آدرس:
```
https://<دامنه-پروژه‌ت>.up.railway.app/admin
```
با همون `ADMIN_USERNAME` و `ADMIN_PASSWORD` که توی Variables گذاشتی وارد شو.
اونجا لیست همه‌ی پیام‌ها، امکان جستجو و فیلتر بر اساس گیرنده رو داری.

## اجرای لوکال (اختیاری، برای تست قبل از دیپلوی)
```bash
npm install
cp .env.example .env
# مقادیر .env رو پر کن، و USE_POLLING=true بذار (چون لوکال webhook عمومی نداری)
npm start
```
پنل مدیریت لوکال: `http://localhost:3000/admin`

## نکات امنیتی و محدودیت‌ها
- حتما `ADMIN_PASSWORD` و `SESSION_SECRET` رو قوی و غیرقابل‌حدس بذار — چون الان پنل ادمین اطلاعات هویتی کاربرها رو هم نشون میده، دسترسی به اون باید فقط دست خودت باشه.
- هویت فرستنده (آیدی عددی، یوزرنیم، نام) فقط توی پنل مدیریت (پشت لاگین) قابل مشاهده‌ست و **هیچوقت به گیرنده‌ی پیام نشون داده نمیشه** — گیرنده همچنان پیام رو کاملا ناشناس می‌بینه.
- این اطلاعات صرفا برای رسیدگی به گزارش مزاحمت/شکایته؛ توصیه میشه فقط در همون چارچوب ازش استفاده کنی.
- آیدی اختصاصی (`custom_id`) در دیتابیس یکتاست؛ اگه کاربری آیدیش رو عوض کنه، لینک قبلیش دیگه کار نمی‌کنه.
- این نسخه روی یک instance ساده طراحی شده (وضعیت «در حال نوشتن پیام» و «در حال ساخت آیدی» توی حافظه نگه داشته میشه)؛ برای اسکیل بزرگ‌تر باید اون بخش رو ببری روی Redis/DB.

## پالت رنگ پنل مدیریت
تم پیش‌فرض تیره با لهجه‌ی بنفش هست، توی `public/admin/style.css` با متغیرهای CSS تعریف شده و راحت قابل تغییره:

| نام متغیر | رنگ | مصرف |
|---|---|---|
| `--bg` | `#0f1115` | پس‌زمینه اصلی |
| `--card` | `#171a21` | پس‌زمینه کارت‌ها و باکس‌ها |
| `--border` | `#262a34` | خطوط و حاشیه‌ها |
| `--text` | `#eef0f4` | متن اصلی |
| `--muted` | `#8b93a3` | متن کم‌رنگ/فرعی |
| `--accent` | `#6d5efc` | رنگ اصلی برند (دکمه‌ها، لینک‌ها) |
| `--accent-hover` | `#7f70ff` | حالت هاور رنگ اصلی |
| `--success` | `#2ecc71` | تایید موفق (مثل «کپی شد») |
| `--warning` | `#f5a623` | هشدار / نشان پرمیوم |
| `--danger` | `#ff5c6c` | خطا |
| `--info` | `#4fb3ff` | اطلاعات (دکمه مشخصات فرستنده) |

اگه تم روشن‌تر می‌خوای، کافیه مقادیر `--bg`, `--card`, `--text` رو عوض کنی؛ بقیه رنگ‌ها (accent/success/warning/danger) روی هر دو تم روشن و تیره خوب دیده میشن.
